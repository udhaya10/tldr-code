Diagnosis (matches forgotten-thread bug class per worker-3-premortem T4):

The Calls handler arm at daemon.rs:727-748 destructures
`DaemonCommand::Calls { path, language: _ }` — the `language` field is
explicitly DISCARDED via `_`. The handler then calls
`build_project_call_graph(&root, Language::Python, None, true)` at L737,
hardcoding Python regardless of what the client requested. A TypeScript-only
project under the Python pipeline produces zero edges because the call-graph
builder filters on `Language::Python`'s file extensions (.py).

The empty `{"edges":[]}` response is the smoking gun: the handler
successfully built a graph but the TS files were invisible to the Python
language pipeline. Post-fix, threading `language.unwrap_or(Language::Python)`
through to `build_project_call_graph` will let the TypeScript pipeline see
the .ts files and produce edges.
